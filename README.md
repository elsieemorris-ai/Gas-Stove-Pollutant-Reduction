# Indoor Air Toolset (Gas Stove Kitchen) 🏠🔥🧪
This tool compares two engineering alternatives for reducing indoor air pollution from residential gas stoves: Alternative A, replacing the gas stove with an electric induction stove, and Alternative B, installing a high-efficiency range hood with increased ventilation. It requires user inputs such as kitchen volume and cooking time, with an assumed baseline air exchange rate, stove power, methane leakage fraction, NOx emission factor, hood capture efficiency, hood flow rate, and electrification fraction. The tool uses a single-zone indoor mass balance model, which is appropriate because it treats the kitchen as a well-mixed space and provides a simple, recognized way to estimate how pollutants build up and are removed during and after cooking. It produces time-series concentration plots, summary tables for CH₄, NO, NO₂, and total NOx, pollutant reduction percentages, exposure estimates, and an MCDA comparison matrix covering technical effectiveness, cost and practicality, public health and safety, and environmental considerations. The comparison is structured so that both alternatives are evaluated under the same baseline kitchen conditions, making it easy to see how each intervention changes emissions and indoor exposure. Users should interpret the outputs by looking first at which option produces the greatest reduction in pollutant concentration and exposure, and then using the MCDA results to weigh those technical benefits against implementation cost, feasibility, health impacts, and broader environmental tradeoffs.
# How to use 🚀
Open the code in the CIVE442Tool2.pynb, and paste it into any platfrom that can read Python code, for example, it was made and tested in Google Colab. Adjust User Inputs: In the second code cell the user must input their kitchen volume in cubic meters (kitchen_vol_m3), cooking time in hours (cooking_time_hr), and post cooking time spent in the kitchen in hours (post_cooking_time_hr). The last section of the user input cell also has MCDA weights that the user may adjust to their liking which control the importance of technical effectiveness, cost, public health, and environmental impacts, but if not there is an assumed weight assigned to each category. After the user has input all data required, they may run the entire code to see results, analysis, and figures.
